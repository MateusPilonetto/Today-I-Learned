Erros

1. Esquecer de fechar tags

A maioria das tags precisa de um par de fechamento. Se você esquecer, o navegador pode "quebrar" o layout da página inteira.

2. IDs duplicados

O atributo id deve ser único na página inteira (como um CPF). Se você quer estilizar vários elementos iguais, use class.

3. Aninhamento incorreto (Cruzamento de tags)

A regra é "O último a entrar é o primeiro a sair". Pense como caixas: você não pode fechar a caixa de fora antes de fechar a caixa de dentro.

4. Colocar elementos de bloco dentro de elementos de linha (especialmente p)

Um erro clássico é colocar uma div dentro de um P. O navegador não aceita isso e automaticamente "expulsa" a div do parágrafo, causando erros visuais estranhos.

5. Esquecer o atributo alt em imagens

O atributo alt (texto alternativo) é obrigatório para acessibilidade (leitores de tela para cegos) e SEO (Google). Se a imagem quebrar, é esse texto que aparece.

6. Usar estilos Inline (style="...")

Evite colocar CSS direto no HTML. Isso torna o código difícil de manter. Sempre prefira usar um arquivo .css separado ou uma tag style.

7. Confundir src com href

É comum trocar as bolas.

  • src (Source/Origem): Usado para incorporar algo na página (img, script, iframe). O navegador "baixa e mostra".
  • href (Hypertext Reference): Usado para apontar para outro lugar (a, link). O navegador "cria uma referência".

8. Esquecer o !DOCTYPE html

Essa deve ser a primeira linha do seu código. Sem ela, o navegador pode entrar em "Quirks Mode" (modo de compatibilidade) e renderizar sua página como se estivéssemos nos anos 90, quebrando seu CSS moderno.

9. Formulários sem label

Todo (input) deve ter um rótulo associado. Isso é vital para saber o que digitar no campo e ajuda na acessibilidade.

10. "Divite" (Excesso de DIVs)

Usar div para tudo. O HTML5 trouxe tags semânticas que explicam o que o conteúdo é. O Google adora tags semânticas.
